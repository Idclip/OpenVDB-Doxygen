OpenVDB AX Dependencies

OpenVDB AX depends on the OpenVDB Core library and thus indirectly requires its dependencies as listed in the OpenVDB Dependency Table. The dependencies listed below are additionally specific to OpenVDB AX.

Dependency Table

Package Minimum Recommended Description apt-get Homebrew Source
LLVM 5.0.0 8.0.0 Target-independent code generation Y Y https://llvm.org/
Boost 1.61 1.61 Components: random Y Y https://www.boost.org
Bison* 3.0.0 3.0.5 General-purpose parser generator Y Y https://www.gnu.org/software/gcc
Flex* 2.6 2.6.4 Fast lexical analyzer generator Y Y https://github.com/westes/flex
  • Note - Bison and Flex are only required for re-generation of the AX grammar which is not typically necessary for the majority of users.

Using UNIX apt-get

The following commands demonstrate how to install OpenVDB AX dependencies on UNIX through apt-get. Note that your chosen llvm version can be selected by replacing the version number in the dev package name.
1 #!/bin/bash
2 apt-get install llvm-8-dev # LLVM 8
3 apt-get install boost-random-dev # Boost::random
4 apt-get install bison # Bison
5 apt-get install flex # Flex

Using Homebrew

The following commands demonstrate how to install OpenVDB AX dependencies through homebrew. Note that your chosen llvm version can be selected by replacing the version number after the @ symbol. If no version number is supplied, homebrew will install the latest version available.
1 #!/bin/bash
2 brew install llvm@8 # LLVM 8
3 brew install boost # Boost
4 brew install bison # Bison
5 brew install flex # Flex

Building OpenVDB AX

OpenVDB AX's build system is heavily based on the existing infrastructure in OpenVDB. To this end, it's encouraged that users wishing to build OpenVDB AX from source read the OpenVDB build documentation which also describes the process of installing OpenVDB and its required dependencies.
In addition, the following CMake variables can be used to enable or disable OpenVDB AX Components.
Component CMake Variable Default
OpenVDB AX Library OPENVDB_BUILD_AX ON
OpenVDB AX Binary OPENVDB_BUILD_AX_BINARIES ON
OpenVDB AX Python OPENVDB_BUILD_AX_PYTHON_MODULE OFF
OpenVDB AX Houdini OPENVDB_BUILD_AX_HOUDINI_PLUGIN OFF
OpenVDB AX Grammar OPENVDB_BUILD_AX_GRAMMAR OFF
OpenVDB AX Unit Tests OPENVDB_BUILD_AX_UNITTESTS OFF
OpenVDB AX Documentation OPENVDB_BUILD_AX_DOCS OFF
A typical build command, from the root directory of the OpenVDB AX repository may look as follows:
1 mkdir build
2 cd build
3 cmake -DOPENVDB_BUILD_AX=ON ../
4 make -j4 install
